ETL (Extraction, Transform, and Load)¶

The aviation accident dataset for 1985 is available in .htm format on the planecrashinfo.com website. To access and scrape this data, I utilize two libraries: requests and beautifulsoup4, which facilitate web page access and HTML document parsing. Additionally, the pandas library is imported for data manipulation and analysis in a tabular format. The function scrape_planecrash_data(url) is designed to retrieve and parse the webpage's content. It sends a GET request to the specified URL using requests.get(url), and the HTML response is processed with BeautifulSoup. The content is parsed using Python's built-in HTML parser (html.parser), with the parsed data stored in the soup variable. Finally, soup.find('table') is used to locate the first <table> tag on the page, which is assumed to contain the relevant accident data.

The column names are loaded into the first row of the table which is resolved by setting the columns to the row with the index [0]. The index is then reset.

import requests
from bs4 import BeautifulSoup
import pandas as pd
def scrape_planecrash_data(url):
    response = requests.get(url)
    soup = BeautifulSoup(response.content, 'html.parser')

    # Find the relevant table (assuming it's the first <table> tag)
    table = soup.find('table')

    # Convert the HTML table to a pandas DataFrame
    df = pd.read_html(str(table))[0]
    return df
url_1985 = 'http://planecrashinfo.com/1985/1985.htm'
df_1985 = scrape_planecrash_data(url_1985)
df_1985.columns = df_1985.iloc[0]
df_1985 = df_1985.drop(0).reset_index(drop=True)

Questions for the Dataset¶

Frequency of Crashes by Month¶

One question to explore with this dataset is the frequency of crashes by month. By grouping the data by month and counting the frequency of crashes for each, I can identify the month with the highest number of crashes using the .idxmax() function. This method can also reveal the month with the fewest crashes in 1985. To obtain the specific number of crashes for the most and least frequent months, the values can be accessed with the month_counts['max_month'] and month_counts['min_month'] functions.

In [6]:
month_counts = df_1985.groupby('Month')['Date'].count()
max_month = month_counts.idxmax()
print(f"The month with the most crashes in 1985 was the {max_month}st month at {month_counts[max_month]}.")
min_month = month_counts.idxmin()
print(f"The month with the least crashes in 1985 was the {min_month}rd month at {month_counts[min_month]}.")

The month with the most crashes in 1985 was the 1st month at 9.
The month with the least crashes in 1985 was the 3rd month at 2.

Survival Rate for Crashes in 1985¶

To determine the survival rate of aviation crashes in 1985, a new column called Survival Rate is created. This column is calculated by subtracting Total Fatalities from Aboard and then dividing the result by Aboard. The mean of these values provides the overall survival rate for crashes in the year 1985. Once calculated, this new column is utilized for visualization, making it easier to represent the survival rates across different incidents.

In [10]:
survival_rate = ((df_1985['Aboard'] - df_1985['Total Fatalities']) / df_1985['Aboard']).mean()
print(f"The survival rate for crashes in 1985 was {survival_rate :.2%}.")

The survival rate for crashes in 1985 was 13.90%.

Total Fatalities by Aircraft Manufacturer¶

To analyze the total fatalities associated with aviation accidents by aircraft manufacturer, the groupby() function can be utilized. First, the function groups the data by the Aircraft Manufacturer column, aggregating all records related to each manufacturer together. Next, it calculates the sum of the Total Fatalities column for each group, resulting in the total number of fatalities associated with crashes for each aircraft manufacturer. Finally, the sums are sorted in descending order using the sort_values(ascending=False) method, allowing for easy identification of which manufacturers had the highest total fatalities in 1985. The final output is a Series where the index represents the aircraft manufacturers, and the values represent the total fatalities, sorted from the manufacturer with the most fatalities to the one with the least.

III. Data Challenges¶

Issues with the Dataset: Sources¶

The aviation accident dataset for 1985 provides sufficient information for preliminary analysis; however, there are several concerns regarding the accuracy of the source and the amount of missing data. The website PlaneCrashInfo.com includes a disclaimer stating that there is no guarantee of the validity of the information provided, despite efforts to verify accuracy from numerous sources. Furthermore, the information on the site does not claim to reflect the opinions or determinations of national authorities.

Issues with the Dataset: Month and Day Analyses¶

The limited data within the yearly datasets can restrict further analysis. Consequently, conclusions drawn from these visualizations may not be definitive, as many confounding variables are not controlled for. Utilizing datasets from other years could help identify any patterns related to the month, or day, of flight and the associated risk of aviation accidents.

Issues with the Dataset: Aircraft Manufacturer Analysis¶

The analysis of aircraft manufacturers associated with the most fatalities in 1985 lacks context. These statistics do not account for the total number of flights conducted for each manufacturer during that year. For instance, while Boeing aircraft may be linked to the highest number of fatalities, this could be primarily due to their significant presence in the overall flight market.

Issues with the Dataset: Missing Data¶

Another challenge is the difficulty in determining the causes of aviation accidents from the available dataset. While there is a summary for each crash, further research or supplementary datasets are necessary to comprehensively compare causes and severity. To conduct any meaningful analysis regarding relationships over time, access to multiple datasets spanning several years is essential.
